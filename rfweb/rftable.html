The table shows the current associations of VMs and datapaths in the RouteFlow network.

We can have the following types of entries:

  • (VM ID, VM port, -, -, -, -, -)
    Represents a registered and idle VM port.
  • (-, -, CT ID, DP ID, DP port, -, -)
    Represents a registered and idle DP port in a controller.
  • (VM ID, VM port, CT ID, DP ID, DP port, -, -)
    Represents an associated but unmapped entry. This means that RouteFlow is waiting for a mapping message from the VM to the controller in order to know how it's connected to the RouteFlow virtual switch.
  • (VM ID, VM port, CT ID, DP ID, DP port, VS ID, VS port)
    Represents a fully associated and active entry. This means that RouteFlow can redirect routing packets correctly between the datapath and the VM using its virtual switch (RFVS).
